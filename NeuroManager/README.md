5) Детальное описание компонентов:

1. Загрузка и обработка базы знаний

Формат: .docx файл
Обработка: Автоматическое извлечение разделов и подразделов
Категории: Технологии, Этапы, Проблемы, Чек-листы, Общее
2. Векторное хранилище

Embeddings: SentenceTransformers (all-MiniLM-L6-v2)
База данных: ChromaDB с персистентным хранением
Поиск: MMR (Maximal Marginal Relevance) для разнообразия результатов
3. Поисковая система

Основной поиск: Семантический поиск по векторным эмбеддингам
Резервный поиск: Ручной поиск по ключевым словам
Фильтрация: По категориям и релевантности
4. Генерация ответов

Модель: GPT-3.5-turbo через Vsegpt API
Стратегия: Строгое следование контексту из базы знаний
Защита от галлюцинаций: Множественные проверки и ограничения
5. Пользовательский интерфейс

Фреймворк: Gradio ChatInterface
Примеры: Предзагруженные примеры запросов
Категоризация: Автоматическое определение темы вопроса

6) Технологический стек

Python 3.9+
Gradio - веб-интерфейс
LangChain - RAG инфраструктура
ChromaDB - векторная база данных
SentenceTransformers - эмбеддинги
OpenAI API (через Vsegpt) - генерация ответов
python-docx - обработка документов

7) Защита от галлюцинаций

Строгая привязка к контексту документа
Проверка релевантности найденной информации
Четкие сообщения об отсутствии информации
Отказ от ответов на вопросы вне компетенции

8) Процесс обработки запроса

1. 📥 Получение запроса
   │
2. 🏷️ Категоризация запроса
   │   (технологии, этапы, проблемы, чеклисты)
   │
3. 🔍 Поиск в векторной БД
   │   (семантический + резервный по ключевым словам)
   │
4. 📄 Формирование контекста
   │   (фильтрация по категории и релевантности)
   │
5. 🧠 Генерация ответа
   │   (строго по контексту, без выдумывания)
   │
6. ✅ Проверка ответа
   │   (на галлюцинации и релевантность)
   │
7. 📤 Форматирование и вывод

9) Мониторинг и логирование

Система выводит подробные логи в консоль:

- Загруженные разделы документа
- Результаты поиска
- Категоризация запросов
- Статистика ответов
